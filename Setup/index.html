Install Docker Desktop

Option A: Automated Script (Recommended)

Open Terminal and run:

cd /path/to/codeigniter-docker
chmod +x scripts/mac/setup.sh
./scripts/mac/setup.sh

The script installs Homebrew (if needed), Docker Desktop, copies the environment file, and builds all containers automatically.

Option B: Manual Installation

  1. Install via Homebrew: 
    brew install --cask docker
    Or download from docs.docker.com.
  2. Start Docker Desktop from Applications. Wait for the whale icon to stop animating, then verify: 
    docker info
  3. Navigate to the project and set up: 
    cd /path/to/codeigniter-docker
cp .env.example .env
  4. Build and start containers: 
    docker compose up -d --build

    First build: 3–5 minutes. Subsequent starts are near-instant.

  5. Verify: Open http://localhost:8080. You should see a green status page.
Note: On Apple Silicon (M1/M2/M3/M4), all images run natively on ARM — no Rosetta needed.

Install Docker Desktop

Option A: Automated Script (Recommended)

Open Command Prompt or Windows Terminal:

cd C:\path\to\codeigniter-docker
scripts\windows\setup.bat

Or use PowerShell:

powershell -ExecutionPolicy Bypass -File scripts\windows\setup.ps1

Option B: Manual Installation

  1. Enable WSL 2: Open PowerShell as Administrator: 
    wsl --install
    Restart your computer.
  2. Install Docker Desktop: Download from docs.docker.com. Check "Use WSL 2 instead of Hyper-V". Restart if prompted.
  3. Start Docker Desktop from the Start menu. Wait for the green "running" indicator, then verify: 
    docker info
  4. Navigate and set up: 
    cd C:\path\to\codeigniter-docker
copy .env.example .env
  5. Build and start: 
    docker compose up -d --build
  6. Verify: Open http://localhost:8080.
Important: If you see virtualization errors, enable VT-x / AMD-V in your BIOS/UEFI settings.

echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

  containerd.io docker-buildx-plugin docker-compose-plugin
  2. Allow Docker without sudo:

  3. Enable and start:

Install CodeIgniter 4

With the containers running, install CodeIgniter inside the PHP container:

  1. Shell into the PHP container: 
    docker compose exec php bash
  2. Install via Composer: 
    composer create-project codeigniter4/appstarter .
  3. Exit the container: 
    exit
  4. Refresh http://localhost:8080 — you should see the CodeIgniter 4 welcome page.
Tip: The src/ folder on your computer is mounted into the container. Edits in your code editor appear instantly in the browser.

Configure CodeIgniter for MySQL

  5. Click Create repository. Keep this page open — you'll need the URL in the Sourcetree section.
Tip: Your repository URL will look like: https://github.com/your-username/ai-saas-bootcamp.git

Reference Troubleshooting

Common issues and how to resolve them.

Port 8080 or 3306 already in use

Another application is using the port. Either stop it, or edit docker-compose.yml and change the port mapping (e.g., "8888:80" for Nginx or "33060:3306" for MySQL).

MySQL connection refused

The MySQL container may still be initialising. Wait 15–30 seconds and try again. Check its logs:

docker compose logs mysql

Look for "ready for connections" in the output.

Docker build fails

  • Check your internet connection — Docker downloads base images.
  • Try a clean rebuild: docker compose build --no-cache
  • Ensure at least 2 GB of free disk space.

Sourcetree asks for a password when pushing

GitHub no longer accepts account passwords. Create a Personal Access Token: GitHub → Settings → Developer Settings → Personal Access Tokens → Tokens (classic) → Generate New Token. Give it repo scope. Use this token as your password.

Claude Code: API key not working

  • Ensure your Anthropic account has API credits (console.anthropic.com).
  • Verify the key was copied completely (no trailing spaces).
  • Try setting it as an environment variable and running claude again.

Docker Desktop is slow on macOS

In Docker Desktop preferences → Resources, allocate at least 4 GB of RAM. Under General, enable VirtioFS for faster volume performance.

Virtualisation errors on Windows

Docker requires hardware virtualisation (VT-x / AMD-V). Enter your BIOS/UEFI settings and ensure it's enabled.

WSL 2 not installing

Run Windows Update to ensure you're on the latest version. WSL 2 requires Windows 10 version 1903+ or any Windows 11.

Permission denied for docker commands

sudo usermod -aG docker $USER

Then log out and log back in.

Navicat can't connect to MySQL

  • Make sure Docker containers are running: docker compose ps
  • Confirm you're connecting to localhost:3306 (not the Docker internal hostname mysql — that's only for containers).
  • Verify credentials match your .env file.
